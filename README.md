# CampaignAgent

A multi-agent orchestration system that transforms a single campaign strategy brief into a complete set of channel-ready marketing assets — email sequences, LinkedIn posts & ads, blog drafts — with deterministic UTM tracking and structured task plans.

Built with LangGraph, Claude Opus 4, and Streamlit. Designed for human-in-the-loop workflows where the agent drafts and humans approve.

---

## How It Works

```
Campaign Brief (Markdown / pasted text)
        │
        ▼
┌─────────────────┐
│ Strategy Parser  │  ← Agent 1: extracts structured campaign data
└────────┬────────┘
         ▼
┌─────────────────┐
│ Campaign Planner │  ← Agent 2: channel plan, UTM matrix, task breakdown
└────────┬────────┘
         ▼
┌────────────────────────┐
│  HUMAN CHECKPOINT 1    │  Review strategy, plan, UTMs, timeline
│  Edit / Approve        │  Provide feedback → agents regenerate
└────────┬───────────────┘
         ▼
┌────────┼────────┐
▼        ▼        ▼
Email   Social   Blog     ← Agents 3a/3b/3c (run in parallel)
Writer  Writer   Writer
└────────┼────────┘
         ▼
┌────────────────────────┐
│  HUMAN CHECKPOINT 2    │  Review all draft assets
│  Approve / Regen       │  Per-asset feedback → selective regen
└────────┬───────────────┘
         ▼
┌──────────────────┐
│ Output Assembler  │  ← Agent 4: packages deliverables
└──────────────────┘
         ▼
    Final Output:
    • campaign_assets.md
    • utm_matrix.csv
    • campaign_tasks.json
    • Per-channel files
```

## Features

- **Brief → Assets in minutes** — paste a campaign brief, get a full content package
- **Human-in-the-loop at every stage** — two approval checkpoints with inline editing and feedback
- **Parallel content generation** — email, social, and blog writers run concurrently
- **Deterministic UTM tracking** — all links are generated by a Python function, never by the LLM
- **Selective regeneration** — provide feedback on specific assets without re-running the entire pipeline
- **Brand voice enforcement** — writer agents are loaded with brand guidelines, templates, and examples
- **Structured task output** — JSON task list ready for import into any PM tool (Asana, Linear, Jira)

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Orchestration | [LangGraph](https://github.com/langchain-ai/langgraph) — stateful multi-agent workflows with checkpoints |
| LLM | Claude Opus 4 via [Anthropic API](https://docs.anthropic.com) |
| Interface | [Streamlit](https://streamlit.io) |
| State | LangGraph MemorySaver (dev) / Postgres-ready (prod) |
| UTMs | Custom Python module (`utils/utm.py`) |
| Models | Pydantic v2 + TypedDict (required for LangGraph parallel fan-out) |

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/griffin-arthur/CampaignAgent.git
cd CampaignAgent
```

### 2. Create a virtual environment

```bash
python3.11 -m venv .venv
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -e .
```

### 4. Set your API key

```bash
cp .env.example .env
# Edit .env and add your Anthropic API key
```

### 5. Run the app

```bash
streamlit run app.py
```

The app will open at `http://localhost:8501`.

## Usage

1. **Paste or upload a campaign brief** — use the sample brief (`sample_brief.md`) to test
2. **Review Checkpoint 1** — the parsed strategy and campaign plan are displayed side by side. Edit fields, adjust channels, or provide feedback to regenerate.
3. **Review Checkpoint 2** — all generated content (emails, LinkedIn posts, blog) is shown with per-asset feedback inputs. Approve as-is or request targeted revisions.
4. **Download deliverables** — final output includes combined Markdown assets, a UTM matrix CSV, and a structured JSON task list.

## Agent Details

| Agent | Model | Temp | Purpose |
|-------|-------|------|---------|
| Strategy Parser | Claude Opus 4 | 0.0 | Extract structured data from unstructured brief |
| Campaign Planner | Claude Opus 4 | 0.0 | Generate channel plan, compute UTMs, build task timeline |
| Email Writer | Claude Opus 4 | 0.7 | Draft 3-part email nurture sequence with A/B subject lines |
| Social Writer | Claude Opus 4 | 0.7 | Draft 4 organic LinkedIn posts + 2 sponsored ad variants |
| Blog Writer | Claude Opus 4 | 0.7 | Draft SEO-optimized anchor blog post (1,500–2,000 words) |
| Output Assembler | None (logic only) | — | Format and package all approved assets |

## UTM Generation

UTMs are **deterministic** — generated by a Python function, never by the LLM:

```python
generate_utm(
    campaign_name="finserv-adg-q2-2026",
    source="linkedin",
    medium="social",
    content="organic-post-2"
)
# → https://www.arthur.ai?utm_source=linkedin&utm_medium=social&utm_campaign=finserv-adg-q2-2026&utm_content=organic-post-2
```

The Campaign Planner pre-generates all UTM links. Writer agents embed them — they never construct UTMs themselves.
